Processes & Threads

A process is an address space with resources. Threads are schedulable entities within a process. We'll explore context switching, process creation (fork/exec model on Unix), and lightweight threads (user-level vs kernel-level).

Scheduling Algorithms

Round-robin, priority scheduling, multi-level feedback queue (MLFQ), CFS (Completely Fair Scheduler) in Linux. We'll quantify fairness, latency, throughput, and starvation.

Virtual Memory & Paging

Address translation, page tables (multi-level), TLBs, page replacement algorithms (LRU approximations), and working set model. We'll explain swap, thrashing, and memory-mapped I/O.

Filesystems & Storage Stack

Filesystem layout, inodes, journaling, copy-on-write filesystems (Btrfs, ZFS), and block device layers. We'll examine durability models and metadata performance.

Synchronization & Deadlock

Mutexes, spinlocks, semaphores, condition variables, read-write locks. Deadlock detection and prevention strategies; banker's algorithm for theoretical treatment.

Kernel Architecture

Monolithic vs microkernel designs. Device driver model, system call dispatch, and kernel modules. We'll analyze Linux's VFS, loadable kernel modules, and user-space drivers (FUSE).

Exercises

  1. Simulate scheduling using different algorithms and compare average turnaround time.
  2. Implement a simple page replacement simulator and measure hit rates.
  3. Sketch the steps the kernel takes during process creation and termination.
