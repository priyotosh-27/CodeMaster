Introduction to Operating Systems

An Operating System (OS) is the most crucial piece of software that manages computer hardware and software resources, providing common services for computer programs. It acts as an intermediary between users and the computer hardware.

Core Functions of an Operating System

Process Management

Creates, schedules, and terminates processes. Manages process states (ready, running, waiting, terminated) and handles context switching between processes.

Memory Management

Allocates and deallocates memory for processes. Implements virtual memory, paging, and segmentation to provide efficient memory utilization.

File System Management

Organizes and manages files and directories. Provides file operations, access control, and maintains file system integrity.

Device Management

Controls and coordinates hardware devices through device drivers. Manages I/O operations and device queues.

Types of Operating Systems

Batch Operating Systems

Execute jobs in batches without user interaction. Jobs are collected and processed sequentially. Examples include early IBM mainframe systems.

Time-Sharing Systems

Allow multiple users to interact with the system simultaneously through terminals. CPU time is divided among users using time slicing.

Real-Time Systems

Provide guaranteed response times for critical applications. Hard real-time systems have strict deadlines, while soft real-time systems have more flexibility.

Distributed Systems

Manage resources across multiple interconnected computers, providing transparency and fault tolerance. Examples include cluster computing and cloud platforms.

Processes & Threads

Understanding Processes

A process is a program in execution, consisting of program code, current activity, program counter, processor registers, and process stack. Each process has its own memory space and resources.

Process States

NEW

Process is being created

READY

Waiting for CPU assignment

RUN

Instructions being executed

Process Control Block (PCB)

The PCB contains all information about a process that the OS needs to manage it effectively:

  • Process ID (PID): Unique identifier for the process
  • Process State: Current state of the process
  • Program Counter: Address of next instruction to execute
  • CPU Registers: Contents of processor registers
  • Memory Management: Page tables, segment tables
  • I/O Status: List of open files, I/O requests

Threads

A thread is a lightweight process - a basic unit of CPU utilization that shares code, data, and OS resources with other threads belonging to the same process.

User-Level Threads

  • • Managed by user-space thread library
  • • Fast context switching
  • • Kernel unaware of threads
  • • Blocking system call blocks entire process

Kernel-Level Threads

  • • Managed directly by OS kernel
  • • True parallelism on multiprocessor systems
  • • Individual thread blocking doesn't affect others
  • • Higher overhead for thread operations

Paging

Paging is a memory management scheme that eliminates external fragmentation by dividing physical memory into fixed-size blocks called frames and logical memory into blocks of the same size called pages.

Page Table Structure

Page Number 20 bits
Offset 12 bits
Total: 32-bit virtual address

Page Table Entry

Frame number (physical address)
Valid/Invalid bit
Protection bits (read/write/execute)
Reference bit
Modify bit

Translation Lookaside Buffer (TLB)

TLB is a high-speed cache that stores recent virtual-to-physical address translations to reduce memory access time.

TLB Hit
Translation found in TLB
~1 clock cycle
TLB Miss
Must access page table
~10-100 clock cycles
Page Fault
Page not in memory
~1,000,000 clock cycles

Page Replacement Algorithms

FIFO (First In, First Out)

Replace the page that has been in memory the longest. Simple but can suffer from Belady's anomaly.

Time Complexity: O(1) | Space Complexity: O(n)

LRU (Least Recently Used)

Replace the page that hasn't been used for the longest time. Good performance but expensive to implement.

Implementation approaches:
• Counter method • Stack method • Approximation algorithms

Second Chance (Clock Algorithm)

Enhanced FIFO that gives pages a "second chance" based on reference bit. Circular queue implementation.

If reference bit = 1: Set to 0, move to next page
If reference bit = 0: Replace this page

File Systems & Storage

A file system provides the methods and data structures used to control how information is stored and retrieved on storage devices.

File System Interface

File Operations

  • Create: Allocate space and create directory entry
  • Open: Search directory, load metadata
  • Read/Write: Transfer data to/from memory
  • Seek: Reposition file pointer
  • Delete: Release space, remove directory entry

File Attributes

  • Name: Human-readable identifier
  • Type: File format information
  • Size: Current file size in bytes
  • Protection: Access control permissions
  • Timestamps: Creation, modification, access times

File Allocation Methods

Contiguous Allocation

Each file occupies a contiguous set of blocks on disk.

Advantages:
  • • Excellent read performance
  • • Simple directory structure
  • • Minimal head movement
Disadvantages:
  • • External fragmentation
  • • Difficult file extension
  • • Need to know file size in advance

Linked Allocation

Each file is a linked list of disk blocks scattered throughout the disk.

Advantages:
  • • No external fragmentation
  • • Dynamic file size
  • • Simple allocation
Disadvantages:
  • • Poor random access
  • • Reliability issues
  • • Space overhead for pointers

Indexed Allocation

Each file has an index block containing pointers to all data blocks of the file.

Advantages:
  • • Good random access
  • • No external fragmentation
  • • Dynamic allocation
Disadvantages:
  • • Index block overhead
  • • Index block size limitation
  • • Need multilevel indexing for large files

Inode Structure (UNIX/Linux)

An inode (index node) contains metadata about a file and pointers to its data blocks.

Direct Pointers (12)
Point directly to data blocks
File size: 0-48KB
Single Indirect
Points to block of pointers
File size: 48KB-4MB
Double Indirect
Two levels of indirection
File size: 4MB-4GB
Triple Indirect
Three levels of indirection
File size: 4GB-4TB

Process Synchronization

Process synchronization ensures correct execution of cooperating processes that share resources or communicate with each other.

Critical Section Problem

A critical section is a code segment where shared variables are accessed. Only one process should execute in its critical section at a time.

Mutual Exclusion

Only one process can be in critical section at a time

Progress

Selection of next process cannot be postponed indefinitely

Bounded Waiting

Limit on number of times others enter before a waiting process

Synchronization Primitives

Mutex (Mutual Exclusion)

A binary lock that allows only one thread to access a shared resource at a time.

mutex_lock():
• Acquire exclusive access
• Block if already locked
mutex_unlock():
• Release exclusive access
• Wake waiting threads

Semaphore

A signaling mechanism with a counter that controls access to shared resources.

Binary Semaphore
  • • Value: 0 or 1
  • • Similar to mutex
  • • Used for mutual exclusion
Counting Semaphore
  • • Value: 0 to N
  • • Counts available resources
  • • Used for resource allocation

Condition Variables

Allow threads to wait for specific conditions to become true and to signal when conditions change.

wait()
Release mutex and block until signaled
signal()
Wake up one waiting thread
broadcast()
Wake up all waiting threads

Deadlocks

A deadlock is a situation where a set of processes are blocked because each process is holding a resource and waiting for another resource acquired by some other process.

Necessary Conditions for Deadlock

Mutual Exclusion

At least one resource must be held in a non-shareable mode. Only one process can use the resource at a time.

Hold and Wait

A process holding at least one resource is waiting to acquire additional resources held by other processes.

No Preemption

Resources cannot be forcibly removed from a process. They must be released voluntarily by the holding process.

Circular Wait

A set of processes {P₀, P₁, ..., Pₙ} exists such that P₀ waits for P₁, P₁ waits for P₂, ..., Pₙ waits for P₀.

Deadlock Handling Strategies

Prevention

Ensure that at least one of the necessary conditions cannot hold:

  • Deny mutual exclusion (when possible)
  • Prevent hold and wait
  • Allow preemption
  • Impose total ordering of resources

Avoidance (Banker's Algorithm)

Dynamically examine resource allocation to ensure system never enters deadlock state:

Safe State:
System can allocate resources to each process in some order and avoid deadlock
Unsafe State:
No guarantee that deadlock can be avoided

Detection and Recovery

Allow deadlocks to occur, detect them, and recover:

Detection Methods:
  • • Wait-for graph
  • • Resource allocation graph
  • • Periodic deadlock detection
Recovery Methods:
  • • Process termination
  • • Resource preemption
  • • Rollback to safe state

Security & Protection

Operating system security involves protecting system resources from unauthorized access, ensuring data integrity, and maintaining system availability.

Access Control Models

Discretionary Access Control (DAC)

Resource owners control access permissions. Common in Unix/Linux systems.

rwx rwx rwx
owner group others

Mandatory Access Control (MAC)

System enforces access policies based on security labels and clearances.

Top Secret > Secret > Confidential > Unclassified

Authentication Mechanisms

1

Something You Know

Passwords, PINs, security questions

2

Something You Have

Smart cards, tokens, mobile devices

3

Something You Are

Biometrics: fingerprints, iris scans, face recognition

Common Security Threats

Malware

Viruses, worms, trojans, ransomware

Buffer Overflow

Exploiting memory management vulnerabilities

Privilege Escalation

Gaining higher access levels illegitimately

Social Engineering

Manipulating users to reveal information

DoS Attacks

Overwhelming system resources

Man-in-the-Middle

Intercepting communications

Security Principles

1

Principle of Least Privilege

Grant minimum necessary permissions

2

Defense in Depth

Multiple layers of security controls

3

Fail-Safe Defaults

Default to secure state when failure occurs

4

Complete Mediation

Check every access attempt

5

Separation of Duties

Divide critical operations among multiple people

6

Open Design

Security through design, not obscurity
