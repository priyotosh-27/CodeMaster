What is Computer Networking?

Computer networking is the practice of connecting two or more computing devices together for the purpose of sharing data, resources, and communication. It forms the backbone of modern digital communication.

Key Components:

  • Nodes (computers, servers, devices)
  • Links (cables, wireless connections)
  • Protocols (rules for communication)
  • Network devices (routers, switches, hubs)

OSI Model

The Open Systems Interconnection (OSI) model is a conceptual framework that describes how different network protocols interact and work together to provide network services.

IP Addresses

An Internet Protocol (IP) address is a unique numerical identifier assigned to each device connected to a network. It serves as the device's address for communication.

IPv4 Format:

192.168.1.1

32-bit address (4 octets, 0-255 each)

IPv6 Format:

2001:0db8:85a3:0000:0000:8a2e:0370:7334

128-bit address (8 groups of 4 hexadecimal digits)

Network Topologies

Network topology refers to the physical or logical arrangement of network devices and connections. Different topologies offer various advantages and use cases.

Bus Topology Linear arrangement
Star Topology Central hub connection
Ring Topology Circular connection
Mesh Topology Multiple interconnections

MAC Addresses

A Media Access Control (MAC) address is a unique identifier assigned to network interface cards (NICs) by the manufacturer. It operates at the Data Link Layer (Layer 2).

Format:

00:1A:2B:3C:4D:5E

48-bit address (6 pairs of hexadecimal digits)

Key Features:

  • Unique worldwide identifier
  • Burned into hardware (ROM)
  • First 24 bits = OUI (Organizationally Unique Identifier)
  • Last 24 bits = Device identifier

Subnetting Basics

Subnetting is the practice of dividing a network into smaller sub-networks (subnets). This improves network performance, security, and management by creating logical divisions.

Example Subnet:

192.168.1.0/24

/24 means first 24 bits are network portion

Subnet Mask:

255.255.255.0

Determines network vs host portion

Benefits: Better security, reduced broadcast traffic, improved performance, easier network management

Virtual LANs (VLANs)

VLANs allow network administrators to logically segment a physical network into multiple virtual networks. Each VLAN acts as a separate broadcast domain, providing isolation and improved security.

Key Benefits:

  • Security Enhancement:

    Isolates traffic between different departments or user groups

  • Broadcast Control:

    Reduces broadcast traffic and improves network performance

  • Flexibility:

    Easy to reorganize network structure without physical changes

VLAN Configuration Example:

VLAN 10: Sales Department
IP Range: 192.168.10.0/24
VLAN 20: IT Department
IP Range: 192.168.20.0/24
VLAN 30: Guest Network
IP Range: 192.168.30.0/24

Inter-VLAN Routing: Required for communication between VLANs, typically handled by Layer 3 switches or routers.

Network Address Translation (NAT)

NAT is a method of remapping IP address space by modifying network address information in packet headers while in transit across a traffic routing device.

Types of NAT:

Static NAT (1:1)

One private IP maps to one public IP permanently

Dynamic NAT (Pool)

Multiple private IPs share a pool of public IPs

PAT/NAT Overload (Many:1)

Multiple private IPs share one public IP using port numbers

Routing Protocols

Routing protocols are algorithms used by routers to determine the best paths for forwarding network traffic. They enable dynamic route discovery and maintenance in complex networks.

Distance Vector Protocols

Routers share their entire routing table with neighbors periodically. They calculate the best path based on distance (hop count) and vector (direction).

RIP (Routing Information Protocol)
  • • Maximum 15 hops
  • • Updates every 30 seconds
  • • Simple configuration
  • • Slow convergence
EIGRP (Enhanced Interior Gateway Routing Protocol)
  • • Cisco proprietary
  • • Fast convergence
  • • Supports VLSM
  • • Advanced distance vector

Link State Protocols

Routers build a complete topology map of the network. They use Dijkstra's shortest path algorithm to calculate the best routes.

OSPF (Open Shortest Path First)
  • • Open standard
  • • Hierarchical design (Areas)
  • • Fast convergence
  • • Supports authentication
IS-IS (Intermediate System to Intermediate System)
  • • ISO standard
  • • Used by ISPs
  • • Supports large networks
  • • Dual IS-IS for IPv4/IPv6

BGP (Border Gateway Protocol)

The de facto exterior gateway protocol used to exchange routing information between different autonomous systems (AS) on the Internet.

Quality of Service (QoS)

QoS is a set of technologies and techniques used to manage network traffic and ensure reliable performance for critical applications by controlling bandwidth, latency, and packet loss.

Key QoS Mechanisms:

Traffic Classification:

Identifying different types of traffic (voice, video, data)

Traffic Shaping:

Controlling the rate at which traffic is sent

Queue Management:

Prioritizing packets based on importance

Traffic Priority Classes:

Voice (VoIP) Highest

Low latency, low jitter required

Latency: <150ms, Jitter: <30ms
Video Streaming High

Bandwidth intensive, delay sensitive

Bandwidth: High, Latency: <400ms
Business Applications Medium

Important business traffic

Guaranteed minimum bandwidth
Web Browsing Normal

Standard internet traffic

Best effort delivery

TCP vs UDP Comparison

TCP (Transmission Control Protocol)

Connection-oriented protocol that provides reliable, ordered, and error-checked delivery of data.

Advantages:
  • Reliable delivery
  • Error detection & correction
  • Flow control
  • Congestion control
Disadvantages:
  • No reliability guarantee
  • No error correction
  • No flow control
  • Packets may be lost/duplicated
Common Use Cases:
DNS Queries Video Streaming Online Gaming VoIP DHCP

HTTP vs HTTPS

HTTP (HyperText Transfer Protocol)

Application layer protocol used for transmitting web pages and other resources over the internet.

HTTP Methods:
GET Retrieve data
POST Submit data
PUT Update resource
DELETE Remove resource
Status Codes:
200 OK Successful request
404 Not Found Resource doesn't exist
500 Internal Server Error Server error

HTTPS (HTTP Secure)

HTTP over TLS/SSL providing encryption, data integrity, and authentication.

Security Features:
  • Encryption: Data is encrypted in transit using TLS/SSL
  • Authentication: Verifies server identity with certificates
  • Integrity: Prevents data tampering during transmission
TLS Handshake Process:
1 Client Hello (supported ciphers, random number)
2 Server Hello (chosen cipher, certificate, random number)
3 Client verifies certificate and generates session keys
4 Encrypted communication begins

Domain Name System (DNS)

DNS is the internet's phone book, translating human-readable domain names into IP addresses that computers use to communicate.

DNS Resolution Process

1
User types URL

Browser checks local cache first

2
Recursive Resolver

ISP's DNS server receives the query

3
Root Nameserver

Points to TLD nameserver (.com, .org, etc.)

4
TLD Nameserver

Points to authoritative nameserver

5
Authoritative Server

Returns the IP address

DNS Record Types

A IPv4 Address

Maps domain to IPv4 address (e.g., 192.168.1.1)

AAAA IPv6 Address

Maps domain to IPv6 address

CNAME Canonical Name

Creates alias pointing to another domain

MX Mail Exchange

Specifies mail servers for domain

TXT Text Record

Stores arbitrary text data

DNS Security

DNSSEC (DNS Security Extensions)

Provides authentication and integrity for DNS responses using digital signatures.

  • Prevents DNS spoofing
  • Verifies response authenticity
  • Chain of trust from root
DNS over HTTPS (DoH) / DNS over TLS (DoT)

Encrypts DNS queries to prevent eavesdropping and manipulation.
