Introduction to Java

Get acquainted with the fundamentals of Java, its history, and how to set up your development environment to start coding.

What is Java?

Java is a high-level, class-based, object-oriented programming language that is designed to have as few implementation dependencies as possible. It is a general-purpose programming language intended to let application developers "write once, run anywhere" (WORA), meaning that compiled Java code can run on all platforms that support Java without the need for recompilation. Java applications are typically compiled to bytecode that can run on any Java Virtual Machine (JVM) regardless of the underlying computer architecture.

Why Learn Java?

Java has been a mainstay in the software development world for decades, and for good reason. Here are a few key advantages:

Platform Independent

Thanks to the JVM, Java code can run on any device, from servers and desktops to mobile phones, making it incredibly versatile.

Object-Oriented

Java's strict OOP paradigm helps you build modular, reusable, and maintainable code, which is essential for large-scale applications.

Rich API & Ecosystem

Java has a massive standard library (API) and a rich ecosystem of frameworks like Spring and Hibernate, accelerating development.

High Demand in Industry

Java is a top language for enterprise-level applications, big data, Android development, and scientific computing, ensuring strong job prospects.

Setting up Your Environment

To start writing Java, you need two main things: the Java Development Kit (JDK) and an Integrated Development Environment (IDE). The JDK includes the Java Runtime Environment (JRE) for running code and the compiler (javac) for compiling it.

1. Install the JDK

We recommend installing an LTS (Long-Term Support) version like Java 17 or Java 21. You can download it from Oracle or use an open-source distribution like Eclipse Adoptium.

2. Choose an IDE

An IDE makes coding much easier with features like syntax highlighting, code completion, and debugging tools. Popular choices include:

  • Visual Studio Code (with the Extension Pack for Java) - Lightweight and versatile.
  • IntelliJ IDEA (Community Edition is free) - Powerful and feature-rich.
  • Eclipse IDE for Java Developers - A classic and robust choice.

3. Verify Your Installation

Once installed, open your terminal or command prompt and run the following commands to ensure everything is set up correctly.

Java Core Concepts

Master the building blocks of the Java language. These fundamental concepts are essential for writing any Java program.

Your First Java Program: "Hello, World!"

Let's start with the classic "Hello, World!" program. This simple program will introduce you to the basic syntax and structure of a Java application.

HelloWorld.java
// This is a single-line comment. Every Java program must have a class definition.
public class HelloWorld {

    /* * This is a multi-line comment. The main method is the entry point
     * of any Java application. The JVM starts execution from here.
     */
    public static void main(String[] args) {
        // System.out.println() is used to print text to the console.
        System.out.println("Hello, World!");
    }
}

To run this code:
1. Save it as a file named HelloWorld.java.
2. Open your terminal, navigate to the directory, and compile it: javac HelloWorld.java.
3. This creates a HelloWorld.class file. Now, run it: java HelloWorld.
4. You should see "Hello, World!" printed to your console.

Variables and Data Types

Variables are containers for storing data values. In Java, each variable must have a specified data type. There are two main groups of data types:

  • Primitive Types: These are the most basic data types and are predefined by the language.
  • Reference/Object Types: These refer to objects and are created using defined classes. Examples include String, Array, and any custom classes you create.

Primitive Data Types

Type Size Description
byte 1 byte Stores whole numbers from -128 to 127
short 2 bytes Stores whole numbers from -32,768 to 32,767
int 4 bytes Stores whole numbers from -2,147,483,648 to 2,147,483,647 (most common for integers)
long 8 bytes Stores very large whole numbers. Must end with an 'L' (e.g., 12345L).
float 4 bytes Stores fractional numbers. Sufficient for 6-7 decimal digits. Must end with an 'f' (e.g., 3.14f).
double 8 bytes Stores fractional numbers. Sufficient for ~15 decimal digits (most common for decimals).
boolean 1 bit Stores true or false values.
char 2 bytes Stores a single character/letter or ASCII values. Surrounded by single quotes (e.g., 'A').
DataTypesExample.java
public class DataTypesExample {
    public static void main(String[] args) {
        // Declaring and initializing variables
        int age = 30;
        double price = 19.99;
        char initial = 'J';
        boolean isLoggedIn = true;
        
        // Reference type: String
        String name = "John Doe";
        
        System.out.println("Name: " + name);
        System.out.println("Age: " + age);
        System.out.println("Price: $" + price);
        System.out.println("Initial: " + initial);
        System.out.println("Logged In: " + isLoggedIn);
    }
}

Control Flow

Control flow statements regulate the order in which the program executes instructions.

Conditional Statements (if-else, switch)

ConditionalExample.java
public class ConditionalExample {
    public static void main(String[] args) {
        int score = 85;
        
        // if-else if-else statement
        if (score >= 90) {
            System.out.println("Grade: A");
        } else if (score >= 80) {
            System.out.println("Grade: B");
        } else {
            System.out.println("Grade: C or below");
        }
        
        int dayOfWeek = 3;
        String dayName;

        // switch statement
        switch (dayOfWeek) {
            case 1: dayName = "Monday"; break;
            case 2: dayName = "Tuesday"; break;
            case 3: dayName = "Wednesday"; break;
            default: dayName = "Some other day";
        }
        System.out.println("Today is " + dayName);
    }
}

Loops (for, while, do-while)

LoopExample.java
public class LoopExample {
    public static void main(String[] args) {
        // for loop: executes a block of code a specific number of times.
        System.out.println("--- For Loop ---");
        for (int i = 1; i <= 5; i++) {
            System.out.println("Iteration " + i);
        }
        
        // while loop: executes as long as a condition is true.
        System.out.println("\n--- While Loop ---");
        int count = 1;
        while (count <= 3) {
            System.out.println("Count is " + count);
            count++;
        }
        
        // do-while loop: similar to while, but always executes at least once.
        System.out.println("\n--- Do-While Loop ---");
        int num = 6;
        do {
            System.out.println("Num is " + num);
            num++;
        } while (num <= 5); // Condition is false, but it ran once.
    }
}

Arrays

Arrays are used to store multiple values of the same type in a single variable. The size of an array is fixed once it is created.

ArrayExample.java
public class ArrayExample {
    public static void main(String[] args) {
        // Declare and initialize an array of integers
        int[] numbers = {10, 20, 30, 40, 50};
        
        // Access an element by its index (starts from 0)
        System.out.println("First element: " + numbers[0]); // Output: 10
        System.out.println("Third element: " + numbers[2]); // Output: 30
        
        // Change an element's value
        numbers[1] = 25;
        System.out.println("New second element: " + numbers[1]); // Output: 25
        
        // Get the length of the array
        System.out.println("Array length: " + numbers.length); // Output: 5
        
        // Loop through an array using an enhanced for loop (for-each)
        System.out.println("\nIterating through the array:");
        for (int number : numbers) {
            System.out.print(number + " ");
        }
        // Output: 10 25 30 40 50
    }
}

Methods

A method is a block of code that runs only when it is called. You can pass data, known as parameters, into a method. Methods are used to perform certain actions, and they are also known as functions.

MethodExample.java
public class MethodExample {
    
    // The main method, where the program starts
    public static void main(String[] args) {
        // Call the sayHello method
        sayHello("Alice");
        sayHello("Bob");
        
        // Call the add method and store the returned value
        int sum = add(5, 7);
        System.out.println("The sum is: " + sum);
    }
    
    /**
     * A simple method that takes a name and prints a greeting.
     * This method has a 'void' return type, meaning it doesn't return any value.
     * @param name The name to greet.
     */
    public static void sayHello(String name) {
        System.out.println("Hello, " + name + "!");
    }
    
    /**
     * A method that takes two integers and returns their sum.
     * This method has an 'int' return type.
     * @param a The first number.
     * @param b The second number.
     * @return The sum of a and b.
     */
    public static int add(int a, int b) {
        return a + b;
    }
}

Object-Oriented Programming (OOP)

OOP is the core paradigm of Java. It organizes software design around data, or objects, rather than functions and logic. Understanding these concepts is crucial for writing effective Java code.

Classes and Objects

Think of a class as a blueprint for creating objects. A class defines attributes (fields or variables) and behaviors (methods). An object is an instance of a class. For example, you could have a Car class (the blueprint) and create multiple car objects like myBmw and yourTesla.

Car.java
// This is the blueprint (Class) for a Car
public class Car {
    // Attributes (fields)
    String color;
    String model;
    int year;

    // Behavior (method)
    public void startEngine() {
        System.out.println("The " + color + " " + model + " engine starts. Vroom!");
    }
}
Garage.java (to test the Car class)
public class Garage {
    public static void main(String[] args) {
        // Create an instance (Object) of the Car class
        Car myCar = new Car();
        
        // Set the attributes of the myCar object
        myCar.color = "Red";
        myCar.model = "Ferrari";
        myCar.year = 2023;
        
        // Call the method of the myCar object
        myCar.startEngine(); // Output: The Red Ferrari engine starts. Vroom!

        // Create another Car object
        Car friendsCar = new Car();
        friendsCar.color = "Blue";
        friendsCar.model = "Honda";
        friendsCar.startEngine(); // Output: The Blue Honda engine starts. Vroom!
    }
}

The Four Pillars of OOP

1. Encapsulation

The bundling of data (attributes) and methods that operate on the data into a single unit (a class). It restricts direct access to some of an object's components, which is a means of preventing unintended interference. This is typically achieved using private access modifiers and public "getter" and "setter" methods.

2. Inheritance

A mechanism where a new class (subclass or child class) inherits attributes and methods from an existing class (superclass or parent class). This promotes code reuse. The extends keyword is used to achieve inheritance.

3. Polymorphism

Literally "many forms," it's the ability of an object to take on many forms. The most common use of polymorphism in OOP occurs when a parent class reference is used to refer to a child class object. It allows you to perform a single action in different ways (e.g., through method overriding).

4. Abstraction

The concept of hiding the complex implementation details and showing only the essential features of the object. It helps in reducing programming complexity and effort. This is achieved using abstract classes and interfaces.

Example: Encapsulation

Here, the balance is private. It can only be accessed or modified through the public getBalance() and deposit() methods, giving us control over how the data is handled.

BankAccount.java
public class BankAccount {
    // The balance is private, hidden from outside classes
    private double balance;

    public BankAccount(double initialBalance) {
        if (initialBalance > 0) {
            this.balance = initialBalance;
        }
    }

    // Public "getter" method to safely read the balance
    public double getBalance() {
        return this.balance;
    }

    // Public "setter" like method to modify the balance
    public void deposit(double amount) {
        if (amount > 0) {
            this.balance += amount;
        }
    }
}

Example: Inheritance & Polymorphism

The Dog and Cat classes inherit from Animal and provide their own implementation of the makeSound() method (Method Overriding). This demonstrates both inheritance and polymorphism.

AnimalExample.java
// Superclass
class Animal {
    public void makeSound() {
        System.out.println("Some generic animal sound");
    }
}

// Subclass inheriting from Animal
class Dog extends Animal {
    @Override // This is an annotation, good practice for overriding
    public void makeSound() {
        System.out.println("Woof woof");
    }
}

// Another subclass
class Cat extends Animal {
    @Override
    public void makeSound() {
        System.out.println("Meow");
    }
}

public class TestPolymorphism {
    public static void main(String[] args) {
        Animal myAnimal = new Animal(); // Animal object
        Animal myDog = new Dog();    // Dog object, referenced by Animal type
        Animal myCat = new Cat();    // Cat object, referenced by Animal type
        
        myAnimal.makeSound(); // Calls Animal's method
        myDog.makeSound();    // Calls Dog's overridden method
        myCat.makeSound();    // Calls Cat's overridden method
    }
}

Advanced Java Topics

Ready to go deeper? These topics cover more complex but powerful features of the Java platform, enabling you to build sophisticated applications.

Exception Handling

An exception is an event that disrupts the normal flow of the program. Java's exception handling mechanism allows you to gracefully manage runtime errors using try, catch, and finally blocks.

ExceptionExample.java
public class ExceptionExample {
    public static void main(String[] args) {
        try {
            // Code that might throw an exception
            int[] numbers = {1, 2, 3};
            System.out.println(numbers[10]); // This will cause an ArrayIndexOutOfBoundsException
        } catch (ArrayIndexOutOfBoundsException e) {
            // Block of code to handle the error
            System.out.println("Error: Array index is out of bounds.");
            System.out.println("Exception message: " + e.getMessage());
        } finally {
            // This block will always execute, whether an exception occurred or not.
            // Useful for cleanup, like closing files or database connections.
            System.out.println("The 'try-catch' block is finished.");
        }
    }
}

Java Collections Framework

The Collections Framework provides a set of interfaces and classes to represent and manage groups of objects. It's a crucial part of the Java API.

  • List: An ordered collection that allows duplicate elements. Common implementations are ArrayList (dynamic array) and LinkedList (doubly-linked list).
  • Set: A collection that cannot contain duplicate elements. Common implementations are HashSet (unordered, uses hash codes) and TreeSet (sorted).
  • Map: An object that maps keys to values. Keys must be unique. Common implementations are HashMap (unordered) and TreeMap (sorted by key).
CollectionsExample.java
import java.util.ArrayList;
import java.util.HashMap;
import java.util.List;
import java.util.Map;

public class CollectionsExample {
    public static void main(String[] args) {
        // List example
        List<String> fruits = new ArrayList<>();
        fruits.add("Apple");
        fruits.add("Banana");
        fruits.add("Apple"); // Duplicates are allowed
        System.out.println("Fruits List: " + fruits);

        // Map example
        Map<String, Integer> studentScores = new HashMap<>();
        studentScores.put("Alice", 95);
        studentScores.put("Bob", 88);
        studentScores.put("Charlie", 92);
        System.out.println("Bob's Score: " + studentScores.get("Bob"));
        System.out.println("All Scores: " + studentScores);
    }
}

Lambda Expressions & Streams API (Java 8+)

Introduced in Java 8, lambdas provide a clear and concise way to represent a method interface using an expression. The Streams API uses lambdas to perform SQL-like operations on collections, enabling a more functional style of programming.

StreamsExample.java
import java.util.Arrays;
import java.util.List;
import java.util.stream.Collectors;

public class StreamsExample {
    public static void main(String[] args) {
        List<String> names = Arrays.asList("Alice", "Bob", "Charlie", "Anna", "Alex");

        // Use a stream to find all names starting with "A", convert them to uppercase,
        // and collect them into a new list.
        
        List<String> result = names.stream() // 1. Create a stream
                .filter(name -> name.startsWith("A")) // 2. Intermediate operation: filter
                .map(String::toUpperCase) // 3. Intermediate operation: map to uppercase
                .collect(Collectors.toList()); // 4. Terminal operation: collect results

        System.out.println(result); // Output: [ALICE, ANNA, ALEX]
    }
}

The Java Ecosystem: Frameworks & Tools

Beyond the core language lies a vast ecosystem of powerful frameworks, libraries, and tools that simplify development and enable you to build complex, production-ready applications efficiently.

Spring Framework

The de facto standard for building enterprise-grade Java applications. Spring provides a comprehensive programming and configuration model. Key Modules:

  • Spring Boot: Radically simplifies building stand-alone, production-grade Spring-based applications. It removes boilerplate configuration.
  • Spring MVC: A robust framework for building web applications following the Model-View-Controller design pattern.
  • Spring Data: Simplifies data access from relational and NoSQL databases.
  • Spring Security: Provides powerful authentication and authorization features.

Hibernate

Hibernate is an Object-Relational Mapping (ORM) framework that simplifies database programming. It maps your Java objects (POJOs - Plain Old Java Objects) to database tables, allowing you to interact with the database using high-level object-oriented idioms instead of writing raw SQL. This greatly improves developer productivity and application maintainability.

Build Tools: Maven & Gradle

Build automation tools are essential for managing a project's lifecycle, including dependencies, compilation, testing, and packaging.

  • Maven: Uses a declarative XML file (pom.xml) and follows a convention-over-configuration approach. It has a huge repository of libraries (Maven Central).
  • Gradle: A more modern and flexible build tool that uses a Groovy or Kotlin-based DSL for its build scripts. It often provides better performance than Maven.
