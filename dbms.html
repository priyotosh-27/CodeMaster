Relational Model & Theory

Relational model formalizes data as relations (tables) with well-defined schemas; operations are expressed in relational algebra. Set semantics, keys, and normalization are foundational.

Normal Forms

1NF, 2NF, 3NF, BCNF — normalization reduces redundancy and update anomalies. We'll discuss trade-offs: normalization vs. performance and when to denormalize.

SQL Deep Dive

Advanced Queries

-- Window functions
SELECT user_id, event_time, ROW_NUMBER() OVER (PARTITION BY user_id ORDER BY event_time) AS seq
FROM events;

-- Recursive CTE (hierarchy)
WITH RECURSIVE deps AS (
  SELECT id, parent_id FROM pkg WHERE id = 1
  UNION ALL
  SELECT p.id, p.parent_id FROM pkg p JOIN deps d ON p.parent_id = d.id
)
SELECT * FROM deps;

Transactions & Isolation levels

ACID properties: Atomicity, Consistency, Isolation, Durability. Isolation levels (Read Uncommitted, Read Committed, Repeatable Read, Serializable) trade correctness vs. concurrency.
