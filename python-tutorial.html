Intro & Tooling

Python is a high-level, interpreted language emphasizing readability and developer productivity. Its ecosystem spans web servers, data science, scripting, automation, and embedded systems. CPython is the reference implementation; there are alternatives (PyPy, Jython, IronPython) that are useful in specific contexts.

Install: Use official installers or package managers. For multiple versions prefer pyenv or system packages. Verify with python --version or python3 --version.
IDE: VS Code, PyCharm, or NeoVim. Enable type checking, formatting (Black), linting (Flake8), and test integration.

Your First Script

# hello.py
if __name__ == "__main__":
    print("Hello, Python!")

# Run: python hello.py
Python scripts can be executed directly. For larger apps, use packages and entry points (setup.cfg / pyproject.toml).

Setup & Versions

Python 3.8+ is recommended for modern features (assignment expressions, positional-only params, typing improvements). Know how to manage multiple versions with pyenv and environment isolation with venv or virtualenv. Use pipx for installing CLI tools globally without polluting interpreter site-packages.

Essential Tools

  • pyenv — manage interpreter versions.
  • python -m venv .venv — create local virtual env.
  • pip / pip-tools / pipx — package management.
  • poetry or flit — modern packaging & publishing.

Syntax & Idioms

Python's syntax is intentionally clear: indentation defines blocks, and there is one obvious way to do many things. Embrace comprehension, context managers, and EAFP (Easier to Ask Forgiveness than Permission) instead of LBYL (Look Before You Leap).

# List comprehension
squares = [x*x for x in range(10) if x % 2 == 0]

# Context manager
with open('data.txt') as f:
    first = f.readline().strip()

# EAFP
try:
    value = data['key']
except KeyError:
    value = default
Follow PEP 8 for style and PEP 257 for docstrings. Use Black for formatting — deterministic and widely adopted.

Built-in Types

Python has simple built-ins (int, float, bool, str) and powerful containers (list, tuple, dict, set). Strings are immutable and sequence types offer common protocols (iterable, sequence, mapping).

Control Flow

Use loops, comprehensions, generator expressions, and concise conditionals. Python supports for, while, if/elif/else, and match (structural pattern matching, Python 3.10+).

# match (PEP 634)
point = (0, 0)
match point:
    case (0, 0):
        origin = True
    case (x, 0):
        on_x = True
    case _:
        pass

# generator expression
s = sum(x*x for x in range(1000))

Functions & Closures

Functions are first-class: pass them, return them, and create closures. Use keyword-only arguments and *args/**kwargs judiciously.

def add(a: int, b: int = 0) -> int:
    return a + b

# closure
def make_multiplier(n):
    def mul(x):
        return x * n
    return mul

times3 = make_multiplier(3)
print(times3(5))  # 15
Annotate with types for clarity and tooling support; they are not enforced at runtime unless using runtime checkers.

OOP & Data Classes

Python supports object-oriented programming with classes, inheritance, descriptors, and metaclasses. For value-like classes prefer @dataclass (Python 3.7+).

from dataclasses import dataclass

@dataclass(frozen=True)
class Point:
    x: float
    y: float

p = Point(3, 4)
print(p.x)
Avoid deep inheritance hierarchies; prefer composition. Use slots if you need to reduce memory per-instance: __slots__.

Typing, Protocols & Gradual Typing

Python has gradual typing via the typing module and tools like Mypy. Use types for maintenance, refactoring, and editor support. Familiarize with TypedDict, Protocol, Union, and Optional.

from typing import Protocol, Iterable

class SupportsLen(Protocol):
    def __len__(self) -> int: ...

def needs_len(x: SupportsLen):
    return len(x)
Types are optional — adopt incrementally. Use pyright or mypy in CI to enforce contracts.

Modules, Packages & Packaging

Structure projects with packages (directories + __init__.py) and keep public API surface small. For packaging prefer pyproject.toml (PEP 621) and tools like Poetry or Flit to build and publish wheels.

# package structure
myapp/
  pyproject.toml
  src/
    myapp/
      __init__.py
      main.py

# entry point in pyproject.toml
[project.scripts]
myapp = "myapp.main:cli"
Avoid relative imports outside packages; tests should import from installed package (use editable install pip install -e .).

Virtual Environments & pyenv

Isolate dependencies per-project. Use python -m venv .venv, activate script, and keep .venv out of VCS. For per-user interpreters use pyenv and for global CLI tools use pipx.

python -m venv .venv
source .venv/bin/activate
pip install -U pip setuptools wheel

Asyncio & Concurrency

Asyncio provides cooperative concurrency via event loop and coroutines. Use async/await for high-concurrency IO-bound tasks. Understand where threads/processes are still valuable.

import asyncio

async def fetch(i):
    await asyncio.sleep(0.1)
    return i * i

async def main():
    res = await asyncio.gather(*(fetch(i) for i in range(10)))
    print(res)

asyncio.run(main())
Use anyio or libraries like httpx and asyncpg for async networking and DB access. Prefer FastAPI for async web backends.

Memory Management & the GIL

CPython uses reference counting + cyclic GC. Understand reference cycles and weakrefs. The Global Interpreter Lock (GIL) prevents multiple native threads from executing Python bytecode simultaneously — limiting CPU-bound multi-threaded scaling on CPython.

import weakref

class Node:
    def __init__(self):
        self.ref = None

n = Node()
w = weakref.ref(n)
print(w())
For CPU-bound parallelism use multiprocessing, C extensions, or run worker processes. For IO-bound concurrency prefer asyncio or threads.

Performance & Profiling

  • Measure with cProfile, pyinstrument, or perf. Visualize with SnakeViz or speedscope.
  • Optimize hotspots: algorithmic improvements, reduce allocations, and avoid costly operations in tight loops.
  • Consider PyPy for long-running workloads with many Python-level allocations; use C/Rust extensions for hot CPU code.
python -m cProfile -o out.prof myapp.py
snakeviz out.prof

C Extensions, Cython & Rust

When Python is not fast enough, implement critical code in C (CPython API), Cython, or Rust (pyo3/maturin). Build wheels for distribution to avoid requiring users to compile native code.

# Example Cython snippet (pyx)
# cdef int add(int a, int b):
#     return a + b
C extensions bypass the GIL if written carefully and can deliver huge speedups for tight loops or numeric work.

Testing with pytest

Use pytest for readable tests, fixtures, and parametric testing. Aim for unit tests, integration tests, and end-to-end tests. Use coverage tools to measure testing gaps.

def add(a, b):
    return a + b

def test_add():
    assert add(2, 3) == 5
Mock external I/O (requests, DB) using pytest-mock or MonkeyPatch. Run tests in CI with fast, reproducible environments.

Web Backends: FastAPI & WSGI

FastAPI + Uvicorn is the modern async stack: fast, type-friendly, and excellent for building APIs. For traditional WSGI apps, Flask and Django remain solid choices.

from fastapi import FastAPI

app = FastAPI()

@app.get("/hello")
async def hello(name: str = "World"):
    return {"msg": f"Hello, {name}"}
Run with uvicorn myapp:app --workers 4. Use Gunicorn+Uvicorn worker class for process workers. Always configure timeouts, limits, and graceful shutdowns.

Data Science: NumPy, Pandas & ML

Python dominates data science. Learn NumPy for vectorized ops, Pandas for tabular data, and libraries like scikit-learn, TensorFlow, and PyTorch for ML. Use environments (conda) for complex native deps.

import numpy as np
import pandas as pd

a = np.arange(10)
df = pd.DataFrame({'x': a, 'y': a**2})
print(df.describe())
Avoid using Pandas in tight loops—vectorize. Be conscious of memory when working with large datasets; consider chunked processing or Dask.

Security & Best Practices

Sanitize inputs, use prepared statements/ORM parameterization, validate and escape data when rendering, and use secure defaults. Keep dependencies up to date and scan for vulnerabilities (Safety, Dependabot).

  • Never store plaintext secrets in code—use environment variables and secret stores.
  • Use HTTPS, proper cookie flags, and CSRF protection for web apps.
  • Limit user-supplied file uploads and validate file types.
