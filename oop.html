üéØ What is Object-Oriented Programming?

Object-Oriented Programming (OOP) is a programming paradigm that organizes code into objects - self-contained units that combine data (attributes) and behavior (methods) together. Think of it like modeling the real world in your code!

üí° Real-World Analogy

Imagine a Car. It has properties like color, model, speed, and actions like start(), stop(), accelerate(). In OOP, we create a Car class that bundles these properties and behaviors together!

üìö Key Concepts

Class

A blueprint or template for creating objects

Object

An instance of a class with actual values

Method

Functions that define what an object can do

Attribute

Variables that store the object's data

üî• Your First Java Class

// Car.java - A simple class example
public class Car {
    // Attributes (Properties)
    private String brand;
    private String color;
    private int speed;
    private boolean isEngineOn;
    
    // Constructor - special method to create objects
    public Car(String brand, String color) {
        this.brand = brand;
        this.color = color;
        this.speed = 0;
        this.isEngineOn = false;
    }
    
    // Methods (Behaviors)
    public void startEngine() {
        isEngineOn = true;
        System.out.println(brand + " engine started! üöó");
    }
    
    public void accelerate(int speedIncrease) {
        if (isEngineOn) {
            speed += speedIncrease;
            System.out.println("Speed increased to " + speed + " km/h");
        } else {
            System.out.println("Start the engine first!");
        }
    }
    
    public void displayInfo() {
        System.out.println("üöó " + color + " " + brand + 
                         " | Speed: " + speed + " km/h" +
                         " | Engine: " + (isEngineOn ? "On" : "Off"));
    }
    
    // Getters and Setters
    public String getBrand() { return brand; }
    public String getColor() { return color; }
    public int getSpeed() { return speed; }
}

// Using the Car class
public class Main {
    public static void main(String[] args) {
        // Creating objects (instances)
        Car myCar = new Car("Toyota", "Red");
        Car friendsCar = new Car("Honda", "Blue");
        
        // Using the objects
        myCar.startEngine();
        myCar.accelerate(60);
        myCar.displayInfo();
        
        friendsCar.displayInfo(); // Engine is off, speed is 0
    }
}

üèõÔ∏è The Four Pillars of OOP

üîí

Encapsulation

Bundle data and methods together, hide internal details, and provide controlled access through public interfaces.

üß¨

Inheritance

Create new classes based on existing ones, inheriting properties and behaviors while adding new features.

üé≠

Polymorphism

Use the same interface for different underlying forms, allowing objects to behave differently based on their type.

üé®

Abstraction

Hide complex implementation details and show only essential features, simplifying the interface for users.

üîí Encapsulation - Data Security & Control

Encapsulation is like having a protective capsule around your data. It bundles data (variables) and methods (functions) into a single unit and restricts direct access to some components.

üéØ Key Benefits

  • Data Protection: Prevent unauthorized access and modification
  • Controlled Access: Use getters/setters to validate data
  • Modularity: Changes to internal implementation don't affect external code
  • Maintainability: Easier to debug and modify code

üîê Access Modifiers in Java

private

Accessible only within the same class

protected

Accessible within package and subclasses

public

Accessible from anywhere

default

Accessible within the same package

üí° Practical Example: Bank Account

public class BankAccount {
    // Private attributes - cannot be accessed directly
    private String accountNumber;
    private double balance;
    private String ownerName;
    private boolean isActive;
    
    // Constructor
    public BankAccount(String accountNumber, String ownerName, double initialBalance) {
        this.accountNumber = accountNumber;
        this.ownerName = ownerName;
        this.balance = initialBalance >= 0 ? initialBalance : 0; // Validation
        this.isActive = true;
    }
    
    // Public methods to access private data (Getters)
    public String getAccountNumber() {
        return accountNumber;
    }
    
    public double getBalance() {
        return balance;
    }
    
    public String getOwnerName() {
        return ownerName;
    }
    
    public boolean isAccountActive() {
        return isActive;
    }
    
    // Controlled modification methods
    public boolean deposit(double amount) {
        if (!isActive) {
            System.out.println("‚ùå Account is inactive!");
            return false;
        }
        
        if (amount <= 0) {
            System.out.println("‚ùå Deposit amount must be positive!");
            return false;
        }
        
        balance += amount;
        System.out.println("‚úÖ Deposited $" + amount + ". New balance: $" + balance);
        return true;
    }
    
    public boolean withdraw(double amount) {
        if (!isActive) {
            System.out.println("‚ùå Account is inactive!");
            return false;
        }
        
        if (amount <= 0) {
            System.out.println("‚ùå Withdrawal amount must be positive!");
            return false;
        }
        
        if (amount > balance) {
            System.out.println("‚ùå Insufficient funds! Available: $" + balance);
            return false;
        }
        
        balance -= amount;
        System.out.println("‚úÖ Withdrew $" + amount + ". New balance: $" + balance);
        return true;
    }
    
    // Internal method to deactivate account
    public void deactivateAccount() {
        isActive = false;
        System.out.println("üîí Account deactivated!");
    }
    
    // Display account info
    public void displayAccountInfo() {
        System.out.println("üë§ Account: " + accountNumber);
        System.out.println("üë§ Owner: " + ownerName);
        System.out.println("üí∞ Balance: $" + balance);
        System.out.println("üîÑ Status: " + (isActive ? "Active" : "Inactive"));
        System.out.println("‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ");
    }
}

üß¨ Inheritance - Code Reusability

Inheritance allows you to create new classes based on existing ones. The new class inherits properties and methods from the parent class, and can add its own unique features.

üå≥ Family Tree Analogy

Just like you inherit traits from your parents (eye color, height), classes can inherit features from parent classes. A Dog class might inherit from Animal, getting basic animal behaviors while adding dog-specific ones like barking.

üéØ Types of Inheritance

Single

One child class inherits from one parent class

Multilevel

Child class becomes parent to another class

Hierarchical

Multiple child classes inherit from one parent

üöó Complete Inheritance Example: Vehicle System

// Parent class (Base class/Superclass)
public class Vehicle {
    // Protected: accessible by child classes
    protected String brand;
    protected String model;
    protected int year;
    protected double fuelCapacity;
    protected double currentFuel;
    
    // Constructor
    public Vehicle(String brand, String model, int year, double fuelCapacity) {
        this.brand = brand;
        this.model = model;
        this.year = year;
        this.fuelCapacity = fuelCapacity;
        this.currentFuel = 0.0;
    }
    
    // Common methods for all vehicles
    public void startEngine() {
        if (currentFuel > 0) {
            System.out.println("üî• " + brand + " " + model + " engine started!");
        } else {
            System.out.println("‚õΩ No fuel! Please refuel first.");
        }
    }
    
    public void stopEngine() {
        System.out.println("üõë " + brand + " " + model + " engine stopped.");
    }
    
    public void refuel(double amount) {
        if (amount > 0 && (currentFuel + amount) <= fuelCapacity) {
            currentFuel += amount;
            System.out.println("‚õΩ Refueled " + amount + "L. Current fuel: " + currentFuel + "L");
        } else {
            System.out.println("‚ùå Invalid fuel amount or tank would overflow!");
        }
    }
    
    public void displayInfo() {
        System.out.println("üöó Vehicle Info:");
        System.out.println("   Brand: " + brand);
        System.out.println("   Model: " + model);
        System.out.println("   Year: " + year);
        System.out.println("   Fuel: " + currentFuel + "/" + fuelCapacity + "L");
    }
    
    // Method that can be overridden by child classes
    public void move() {
        System.out.println("üöó Vehicle is moving...");
    }
}

// Child class 1: Car extends Vehicle
public class Car extends Vehicle {
    private int numberOfDoors;
    private String transmissionType;
    private boolean hasAirConditioning;
    
    // Constructor - calls parent constructor using super()
    public Car(String brand, String model, int year, double fuelCapacity, 
               int numberOfDoors, String transmissionType) {
        super(brand, model, year, fuelCapacity); // Call parent constructor
        this.numberOfDoors = numberOfDoors;
        this.transmissionType = transmissionType;
        this.hasAirConditioning = true;
    }
    
    // Car-specific methods
    public void openTrunk() {
        System.out.println("üéí Car trunk opened!");
    }
    
    public void toggleAirConditioning() {
        hasAirConditioning = !hasAirConditioning;
        System.out.println("‚ùÑÔ∏è Air conditioning: " + (hasAirConditioning ? "ON" : "OFF"));
    }
    
    public void playMusic() {
        System.out.println("üéµ Car stereo is playing music!");
    }
    
    // Override parent method with car-specific behavior
    @Override
    public void move() {
        if (currentFuel > 0) {
            System.out.println("üöó Car is driving smoothly on the road...");
            currentFuel -= 1.0; // Consume fuel
        } else {
            System.out.println("‚õΩ Car can't move - no fuel!");
        }
    }
    
    // Override to include car-specific info
    @Override
    public void displayInfo() {
        super.displayInfo(); // Call parent method
        System.out.println("   Doors: " + numberOfDoors);
        System.out.println("   Transmission: " + transmissionType);
        System.out.println("   AC: " + (hasAirConditioning ? "Available" : "Not Available"));
    }
}

// Child class 3: Truck extends Vehicle (Multilevel inheritance example)
public class Truck extends Vehicle {
    private double cargoCapacity;
    private double currentCargo;
    
    public Truck(String brand, String model, int year, double fuelCapacity, double cargoCapacity) {
        super(brand, model, year, fuelCapacity);
        this.cargoCapacity = cargoCapacity;
        this.currentCargo = 0.0;
    }
    
    public void loadCargo(double weight) {
        if ((currentCargo + weight) <= cargoCapacity) {
            currentCargo += weight;
            System.out.println("üì¶ Loaded " + weight + "kg. Current cargo: " + currentCargo + "kg");
        } else {
            System.out.println("‚ùå Cannot load - would exceed capacity!");
        }
    }
    
    public void unloadCargo(double weight) {
        if (weight <= currentCargo) {
            currentCargo -= weight;
            System.out.println("üì§ Unloaded " + weight + "kg. Remaining cargo: " + currentCargo + "kg");
        } else {
            System.out.println("‚ùå Cannot unload more than current cargo!");
        }
    }
    
    @Override
    public void move() {
        if (currentFuel > 0) {
            System.out.println("üöõ Truck is moving heavy cargo...");
            currentFuel -= 2.0; // Trucks consume more fuel
        } else {
            System.out.println("‚õΩ Truck can't move - no fuel!");
        }
    }
    
    @Override
    public void displayInfo() {
        super.displayInfo();
        System.out.println("   Cargo: " + currentCargo + "/" + cargoCapacity + "kg");
    }
}

üé≠ Polymorphism - One Interface, Many Forms

Polymorphism means "many forms". It allows objects of different classes to be treated as instances of the same class through inheritance, while each object maintains its own specific behavior.

üé™ Real-World Analogy

Think of a remote control. The "power" button works the same way on TVs, radios, and air conditioners, but each device responds differently. Same interface, different behaviors!

üéØ Types of Polymorphism

Runtime Polymorphism

Method overriding - decision made during execution

Compile-time Polymorphism

Method overloading - decision made during compilation

üêæ Complete Polymorphism Example: Animal Kingdom

// Abstract base class
abstract class Animal {
    protected String name;
    protected int age;
    protected String habitat;
    
    public Animal(String name, int age, String habitat) {
        this.name = name;
        this.age = age;
        this.habitat = habitat;
    }
    
    // Abstract method - must be implemented by child classes
    public abstract void makeSound();
    
    // Abstract method for movement
    public abstract void move();
    
    // Concrete method - same for all animals
    public void sleep() {
        System.out.println("üí§ " + name + " is sleeping peacefully...");
    }
    
    public void eat() {
        System.out.println("üçΩÔ∏è " + name + " is eating...");
    }
    
    // Common display method
    public void displayInfo() {
        System.out.println("üêæ Name: " + name + " | Age: " + age + " | Habitat: " + habitat);
    }
}

// Demonstration class showcasing polymorphism
public class PolymorphismDemo {
    public static void main(String[] args) {
        System.out.println("=== POLYMORPHISM DEMO ===\n");
        
        // Create different animals
        Dog buddy = new Dog("Buddy", 5, "Golden Retriever");
        Cat whiskers = new Cat("Whiskers", 3, true);
        Bird robin = new Bird("Robin", 2, "American Robin", 0.3);
        Fish nemo = new Fish("Nemo", 1, "Coral Reef", true);
        Duck donald = new Duck("Donald", 4, "Mallard");
        
        // Polymorphism in action - same method call, different behaviors
        System.out.println("--- POLYMORPHIC ANIMAL SOUNDS ---");
        Animal[] zoo = {buddy, whiskers, robin, nemo, donald};
        
        for (Animal animal : zoo) {
            animal.displayInfo();
            animal.makeSound();  // Each animal makes its own sound!
            animal.move();       // Each animal moves differently!
            animal.eat();        // Common method for all
            System.out.println();
        }
        
        // Interface polymorphism
        System.out.println("--- FLYING ANIMALS ---");
        Flyable[] flyers = {robin, donald};
        for (Flyable flyer : flyers) {
            flyer.fly();
            flyer.land();
        }
        
        System.out.println("\n--- SWIMMING ANIMALS ---");
        Swimmable[] swimmers = {nemo, donald};
        for (Swimmable swimmer : swimmers) {
            swimmer.swim();
            swimmer.dive();
        }
        
        // Method overloading example (Compile-time polymorphism)
        System.out.println("\n--- METHOD OVERLOADING DEMO ---");
        AnimalCare care = new AnimalCare();
        care.treatAnimal(buddy);
        care.treatAnimal(robin, "wing injury");
        care.treatAnimal(nemo, "fin damage", 30.5);
    }
}

    }
    
    public void treatAnimal(Animal animal, String condition, double medication) {

üé® Abstraction - Hiding Complexity

Abstraction means hiding the complex implementation details and showing only the essential features of an object. It's about creating a simple interface that hides the underlying complexity.

üöó Car Dashboard Analogy

When you drive a car, you use simple controls: steering wheel, pedals, gear stick. You don't need to understand the complex engine mechanics, transmission systems, or electronic circuits. The dashboard abstracts all that complexity!

üéØ Abstract Classes vs Interfaces

Abstract Classes

  • ‚Ä¢ Can have concrete methods
  • ‚Ä¢ Can have constructors
  • ‚Ä¢ Single inheritance only
  • ‚Ä¢ Can have instance variables

Interfaces

  • ‚Ä¢ All methods are abstract (Java 8+ allows default)
  • ‚Ä¢ No constructors
  • ‚Ä¢ Multiple inheritance supported
  • ‚Ä¢ Only constants allowed

// Abstract base class for media players
abstract class MediaPlayer {
    protected String name;
    protected String currentMedia;
    protected boolean isPlaying;
    protected double volume;
    protected double duration;
    protected double currentTime;
    
    public MediaPlayer(String name) {
        this.name = name;
        this.isPlaying = false;
        this.volume = 50.0;
        this.currentTime = 0.0;
    }
    
    // Abstract methods - must be implemented by subclasses
    public abstract void loadMedia(String mediaPath);
    public abstract void play();
    public abstract void pause();
    public abstract void stop();
    
    // Concrete methods - shared by all media players
    public void setVolume(double volume) {
        if (volume >= 0 && volume <= 100) {
            this.volume = volume;
            System.out.println("üîä Volume set to " + volume + "%");
        } else {
            System.out.println("‚ùå Volume must be between 0-100%");
        }
    }
    
    public void displayStatus() {
        System.out.println("üì± " + name + " Status:");
        System.out.println("   Media: " + (currentMedia != null ? currentMedia : "None"));
        System.out.println("   Status: " + (isPlaying ? "Playing ‚ñ∂Ô∏è" : "Stopped ‚èπÔ∏è"));
        System.out.println("   Volume: " + volume + "%");
        if (currentMedia != null) {
            System.out.println("   Time: " + formatTime(currentTime) + "/" + formatTime(duration));
        }
    }
    
    // Helper method
    private String formatTime(double seconds) {
        int minutes = (int) (seconds / 60);
        int secs = (int) (seconds % 60);
        return String.format("%02d:%02d", minutes, secs);
    }
    
    // Template method pattern
    public final void startPlayback(String mediaPath) {
        loadMedia(mediaPath);
        play();
        System.out.println("‚úÖ Playback started!");
    }
}

class AudioPlayer extends MediaPlayer implements AdvancedPlayback {
    private String audioFormat;
    private double playbackSpeed;
    private String[] playlist;
    private int currentTrackIndex;
    
    public AudioPlayer(String name) {
        super(name);
        this.playbackSpeed = 1.0;
        this.currentTrackIndex = 0;
    }
    
    @Override
    public void loadMedia(String mediaPath) {
        this.currentMedia = mediaPath;
        this.audioFormat = detectAudioFormat(mediaPath);
        this.duration = calculateDuration(mediaPath);
        this.currentTime = 0.0;
        System.out.println("üéµ Loaded audio: " + mediaPath + " (" + audioFormat + ")");
    }
    
    @Override
    public void play() {
        if (currentMedia != null) {
            isPlaying = true;
            System.out.println("üéµ Playing: " + currentMedia + " at " + playbackSpeed + "x speed");
        } else {
            System.out.println("‚ùå No media loaded!");
        }
    }
    
    @Override
    public void pause() {
        if (isPlaying) {
            isPlaying = false;
            System.out.println("‚è∏Ô∏è Audio paused");
        }
    }
    
    @Override
    public void stop() {
        isPlaying = false;
        currentTime = 0.0;
        System.out.println("‚èπÔ∏è Audio stopped");
    }
    
    // Implementing AdvancedPlayback interface
    @Override
    public void seekTo(double time) {
        if (currentMedia != null && time >= 0 && time <= duration) {
            currentTime = time;
            System.out.println("‚è© Seeked to " + formatTime(time));
        }
    }
    
    @Override
    public void setPlaybackSpeed(double speed) {
        if (speed > 0 && speed <= 3.0) {
            this.playbackSpeed = speed;
            System.out.println("‚ö° Playback speed: " + speed + "x");
        }
    }
    
    @Override
    public void createPlaylist(String[] mediaPaths) {
        this.playlist = mediaPaths;
        System.out.println("üìú Playlist created with " + mediaPaths.length + " tracks");
    }
    
    public void nextTrack() {
        if (playlist != null && currentTrackIndex < playlist.length - 1) {
            currentTrackIndex++;
            loadMedia(playlist[currentTrackIndex]);
            play();
        }
    }
    
    // Helper methods (hidden complexity)
    private String detectAudioFormat(String path) {
        if (path.endsWith(".mp3")) return "MP3";
        if (path.endsWith(".wav")) return "WAV";
        if (path.endsWith(".flac")) return "FLAC";
        return "Unknown";
    }
    
    private double calculateDuration(String path) {
        // Simulate complex duration calculation
        return 180.0 + Math.random() * 120; // Random duration between 3-5 minutes
    }
    
    private String formatTime(double seconds) {
        int minutes = (int) (seconds / 60);
        int secs = (int) (seconds % 60);
        return String.format("%02d:%02d", minutes, secs);
    }
}

// Thread-safe Singleton implementation
public class DatabaseConnection {

    private boolean isConnected;

    }

        if (instance == null) {
            instance = new DatabaseConnection();

            System.out.println("‚ÑπÔ∏è Already connected");
        }
    }

        if (isConnected) {
            isConnected = false;
            System.out.println("‚ùå Disconnected from database");
        }
    }
    
    public void executeQuery(String query) {
        if (isConnected) {
            System.out.println("üîç Executing: " + query);
        } else {
            System.out.println("‚ùå Not connected to database");

    }
